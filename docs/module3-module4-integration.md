# Module 3 to Module 4 Integration

## Upstream Source

Module 4 reads SIEM alerts generated by Module 3 from:

- `module-3-siem-visualization (2)/module-3-siem-visualization/module-3-siem-visualization/data/generated-alerts.ndjson`

Only records whose `destination_module` contains `Module 4 - Threat Intelligence Enrichment` are imported.

## Processing

1. Module 4 imports Module 3 alert records.
2. IOC-bearing fields are mapped into the Threat Intelligence enrichment schema.
3. Enrichment runs against the IOC registry plus MISP/OpenCTI connectors or mock feeds.
4. Enriched alerts are stored locally.
5. A downstream IR handoff file is written.

## Downstream Output

Module 4 writes enriched IR-ready records to:

- `backend/output/module4-enriched-alerts-for-ir.ndjson`

These records contain:

- original Module 3 rule name
- final severity and risk score
- IOC matches
- MITRE ATT&CK techniques
- recommended response action
- AI classification and anomaly score

## Demonstrable End-to-End Flow

Module 3 dummy alerts:
- `generated-alerts.ndjson`

Module 4 import trigger:
- `POST /api/integrations/module3/import`

Module 4 downstream handoff:
- `backend/output/module4-enriched-alerts-for-ir.ndjson`
